Another remark about the symbol tables: The symbol tables described here
have nothing to do with the symbol tables that can be attached to an
assembled program.

With the menu item "Save" (Control-Q) you can save your source text under
the name specified in the status bar. No file selector is issued for this.
As an extension, it is always attached. The source text is stored in your
own format. As already mentioned, such source texts are not only saved and
loaded faster, they are also shorter than in the ASCII format. If the source
already exists under this name, a dialog box is issued (see Figure 5.9 on
page 53).

"Save As ..." (F9) corresponds to the "Save" menu item, but this time
the file selector is issued.

Something else about your own source text format; All current settings that
affect the source text are saved (and this is what makes this format so
practical). These are: The cursor position, the 10 highlighters, date and
time the source text was created, date and the time of the last change of
the source text, the place of the last change (CONTROL-Z), a marked block,
